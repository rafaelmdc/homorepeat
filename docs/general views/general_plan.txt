# Homorepeat General Views Plan

## Purpose

This directory is now the top-level planning surface for the first browser
viewer family:

1. `Length`
2. `Codon Composition`
3. `Codon Composition x Length`

The goal is to define these viewers as one coherent system rather than as
independent charts. The existing length explorer is the current implementation
baseline, but it is only one slice of the intended viewer architecture.

## Viewer model

Each viewer follows the same 3-tier structure:

- `Tier 1 - Overview`
  - taxonomy-first
  - lineage-aware
  - hex-style and scalable
- `Tier 2 - Browse`
  - ranked or comparison-oriented
  - filterable
  - compact enough for many taxa
- `Tier 3 - Inspect`
  - one taxon, clade, or filtered subset
  - more classical statistical charts

This structure matters more than any single chart choice. The browser should
default to lineage-aware, compact summaries rather than large raw-point views.

## Current baseline

- `Length` is technically implemented at `/browser/lengths/`.
- `Codon Composition` is implemented and frozen at its MVP browser contract on
  `/browser/codon-ratios/`.
- The current length explorer already provides the reusable browser stats
  foundation:
  - `apps/browser/views/stats/lengths.py`
  - `apps/browser/stats/filters.py`
  - `apps/browser/stats/queries.py`
  - `apps/browser/stats/summaries.py`
  - `apps/browser/stats/payloads.py`
  - `templates/browser/repeat_length_explorer.html`
  - `static/js/repeat-length-explorer.js`
- `Codon Composition` and `Codon Composition x Length` should reuse that
  foundation where it still fits, but they should not inherit the obsolete
  scalar codon-ratio contract.

Current codon-composition MVP freeze:

- route continuity remains `/browser/codon-ratios/`
- overview is the current pairwise lineage-ordered `Taxon x Taxon` heatmap
- browse is stacked taxon composition plus the grouped HTML fallback table
- inspect is the branch-scoped aggregate composition view
- the shared taxonomy gutter and lineage-aware ordering are part of the frozen
  contract
- the original `Taxon x Codon` overview target is deferred beyond the MVP

## First-wave priorities

1. keep the current length explorer as the Tier 2 baseline and align it to the
   general-views architecture
2. centralize shared viewer rules and reuse points
3. replace the scalar codon-ratio assumptions with a codon-composition
   contract built on normalized codon-usage rows
4. define one shared `Tier 1 - Overview` shell so every viewer starts from a
   taxonomy-first hex overview, while allowing the codon-composition MVP
   exception to remain frozen for now
5. keep the codon viewers composition-first on the same browser family rather
   than independent statistical pages

## Shared principles

- keep one Django app: `apps.browser`
- preserve `apps.browser.views` as the stable import surface
- prefer the smallest reuse-friendly abstraction over broad framework work
- keep filters normalized through the shared stats layer
- never ship unbounded raw match sets to the browser
- lineage-aware taxon ordering is mandatory for overview pages
- every first-wave viewer should present a taxonomy-first `Tier 1 - Overview`
  shell for continuity, except where the codon-composition MVP explicitly
  freezes a temporary pairwise-overview exception
- codon viewers support all residues in the product, but codon composition
  remains residue-specific by default
- do not change `homorepeat_pipeline` as part of viewer work unless the user
  explicitly approves that boundary change
- assume the existing finalized codon-usage artifacts already contain the
  codon-composition information the browser needs, and import them correctly
- use real biological values first and keep transformed statistical views as
  secondary companion views
- use normalized codon-usage rows as the hot-path codon browser contract rather
  than a single derived scalar

## Deferred viewers

`Purity`, `Position`, and their comparison pages remain second-wave work. They
should reuse the same shared foundation once the first-wave viewer family is
stable.
