PolyGlutamine in Deuterostomes

A pipeline + SQLite database for comparative polyQ analysis across animal genomes.

#bioinformatics #pipeline #python #sqlite

Overview

This project began as an evolutionary study of polyglutamine (polyQ) tracts, but the main outcome is broader: a reproducible pipeline and relational database for discovering and analyzing homorepeats (polyX) across annotated animal genomes.

The workflow retrieves genome/protein annotations, enriches them with taxonomy, runs configurable detection strategies, stores tract- and protein-level features in SQLite, and generates automated summaries and figures for cross-taxa comparison. PolyQ is used as the primary validation case study.

Key outcomes
  • End-to-end pipeline: data retrieval → homorepeat detection → SQLite storage → automated analyses
  • Multi-strategy detection to reduce definition bias (strict / window / BLAST-based)
  • Figures and poster-ready outputs for comparative genomics + evolutionary interpretation
  • Award winning Poster presentation at the 18th edition of IJUP.

Project Context

Why this project

Homorepeats (polyX tracts) are common in proteins and can influence structure, interaction behavior, regulation, and evolutionary dynamics. But they’re hard to study at scale because “finding repeats” is not a single problem—results depend strongly on how you define a tract.

A strict definition (uninterrupted runs) is very reproducible, but it can miss biologically plausible tracts that contain interruptions (substitutions, small insertions, or imperfect repeat structure). More permissive definitions can capture those interrupted regions, but they can also inflate counts by pulling in generic low-complexity sequence.

This becomes a serious problem in comparative and evolutionary analyses. If different datasets—or different tools within a workflow—use different implicit rules (minimum length, allowed interruptions, window thresholds, treatment of ambiguous residues), then cross-taxon comparisons can become unstable: differences may reflect the detection rules more than the biology. In other words, you can end up “measuring the method” instead of the underlying evolutionary pattern.

This project addresses that by building a workflow where repeat detection is explicit, configurable, and reproducible. Instead of committing to one definition, the pipeline supports multiple detection strategies and makes the parameter choices part of the output. That way, results can be compared fairly under the same rules, and sensitivity to definition can be tested rather than ignored.

To keep everything traceable, outputs are stored in a relational database so each detected tract can be linked back to:

  • the genome/protein sequence it came from
  • the taxonomic context (species, lineage grouping)
  • the detection strategy and the parameters used

This design turns repeat discovery into something that can be audited and reproducible. It also makes downstream analysis much cleaner: you can query by taxon, filter by strategy, rerun analyses with different thresholds, and keep all variants comparable without losing track of how a given tract was defined.

Why polyQ in this project

This project was developed during my final-year BSc internship at the Phenotypic Evolition group in i3S – Instituto de Investigação e Inovação em Saúde (supervised by Cristina Vieira and Jorge Vieira), focused on building infrastructure to study how homorepeat (polyX) tracts vary across lineages. Polyglutamine (polyQ) is used as the primary case study because it aligned with the internship’s neurodegeneration research context and target proteins; the workflow itself is designed to generalize to other polyX repeats.

What I built

I developed a database-backed workflow that converts genome/protein annotations into a structured dataset of homorepeat tracts and derived features. While the internship’s case study focused on polyglutamine (polyQ), the core detection layer is configurable and can be applied to other single–amino-acid repeats (polyX) as well.

The system was designed to support large-scale comparative analysis across taxa: data are retrieved automatically, enriched with taxonomy, processed through complementary repeat-detection strategies, and appended into a relational SQLite database for reproducible querying and downstream visualization.

Main Points
  • Relational SQLite database designed for repeat-centric genomics analysis (genomes ↔ sequences ↔ proteins ↔ repeat regions ↔ taxonomy).
  • Automated ingestion + processing pipeline: retrieval from NCBI, metadata enrichment, detection, and database population.
  • Repeat detection module (“Find Poly”) implemented with three complementary strategies (strict / threshold window / BLAST-based) to balance specificity and sensitivity.
  • Analysis + plotting scripts generating summary distributions and comparative figures directly from database queries.

System workflow

Input preparation

The pipeline starts by retrieving reference genome/protein data from NCBI. Each dataset is screened for potential contaminants and standardized so parsing is consistent across taxa. Taxonomy is appended to headers and file naming so every detected tract remains traceable to its source lineage.

Repeat detection: “Find Poly”

“Find Poly” detects candidate polyX regions using three complementary strategies, so definitions are explicit and results are comparable across taxa.

Method 1 — Pure (strict)

Detects high-purity homorepeat tracts: uninterrupted runs (e.g., polyQ), plus near-pure tracts that contain a single intervening residue within an otherwise continuous stretch. This is the most conservative strategy and prioritizes specificity.

Method 2 — Threshold (density / sliding window)

Applies a sliding window rule that allows limited interruptions while maintaining repeat density. For example, a threshold like Q 6/8 requires at least six glutamines within a window of eight amino acids. Detected regions are then extended while the density criterion continues to hold.

Method 3 — BLAST (seed + merge)

Uses a defined-length homorepeat “seed” to search for matches in the reference data. Hits are merged when they overlap or fall within a proximity threshold, which scales with tract length—helpful when repeats appear fragmented into nearby segments.

All detected regions are stored in a SQLite database, linking each tract to its sequence/protein, taxonomy, method, and parameters. Downstream scripts query the database to generate reproducible summaries and comparative plots.

Key results

In the full report, 697 genomes representing 351 deuterostome families were analyzed, producing large but method-dependent sets of matches (70,010 pure, 122,916 threshold, 91,173 BLAST). This confirmed that detection strategy substantially changes which polyQ tracts are recovered and therefore affects downstream evolutionary interpretation.

Many relationships were analyzed from this data, with the strongest (by far) recurring signal was that polyQ length and glutamine codon usage (%CAG vs %CAA) are linked, and that this relationship varies systematically across taxonomic groups.

Codon usage patterns separate major taxonomic groups

A sign-test heatmap analysis of codon-length relationships showed that average CAG percentage across polyQ sizes can distinguish taxonomic groups, with clearer separation between many basal lineages and mammalian groups. Possibly indicating a framework for separating lineages based on these codon differences, however further research is required.

DESCRIBE THE IMAGE

Longer polyQ tracts are more prevalent in higher-order taxa

longer polyQ tracts become more prevalent in higher-order deuterostomes, particularly mammals (and also birds in several comparisons). This pattern is visible across methods, although each method emphasizes different tract architectures.

BLAST-based approach tends to recover longer and less pure / more interrupted tracts, while the pure method is more specific for short, highly continuous tracts. The threshold method sits between these extremes, capturing slightly longer tracts with limited additional noise. Correctly identifying the best

Higher-order taxa may not only accumulate longer repeats, but may also tolerate or retain more structurally complex (interrupted) polyQ architectures — though BLAST hits require careful interpretation due to very probable false positives, and require future careful tuning, to find the best "true" polyQ's.

PolyQ length and %CAG are strongly correlated (lineage-specific trends)

A major analytical result was the repeated association between polyQ tract length and CAG codon usage, identified through regression analyses (including Huber regression). Across many macro-groups, this was the most informative feature pair for capturing biologically meaningful differences.

Some groups display a negative trend (decreasing %CAG with increasing polyQ length, as to be expected if it's solely a stochastic process), while others show a shallow parabolic pattern, suggesting different repeat-expansion/stabilization dynamics across taxa. The strength of the relationship also changes depending on the maximum polyQ length interval and the detection method used.

Codon composition dynamics (CAG vs CAA) may reflect different lineage-specific strategies balancing repeat expansion, instability, and tolerance of longer polyQ tracts.

Practical outcome: a reusable framework for comparative homorepeat analysis

Beyond the biological findings, the project delivered a reusable computational workflow and database framework for high-throughput polyQ analysis. The poster emphasizes that the pipeline can support large-scale, low-effort analyses and can be adapted to other homorepeat types, while the article demonstrates this through automated extraction, storage, and visualization across multiple taxonomic levels and detection criteria.

Possible Next steps

Integrate richer functional and structural annotations

Future versions of the database can be strengthened by linking polyQ tracts to protein-level annotations, including domain architecture, subcellular localization, disorder predictions, and functional categories. Adding this layer would make it possible to ask not only where polyQ tracts occur across taxa, but also in what biological contexts they are maintained, expanded, or interrupted.

This would also help connect evolutionary patterns in codon composition and tract length to potential functional constraints, including protein interaction interfaces, regulatory proteins, and lineage-specific enrichment of repeat-containing proteins.

Improve taxonomic and phylogenetic resolution

The current analyses highlight strong macro-group patterns, but future work can increase resolution at the species levels. A more explicit phylogenetic framework would help distinguish broad lineage trends from clade-specific events and support more rigorous evolutionary interpretation of repeat expansion and codon usage shifts.

Refine detection and benchmarking strategies

From a informatics point of view, because the project already showed that polyQ detection depends strongly on method choice (pure, threshold/window, and BLAST-based approaches), an important next step is systematic benchmarking. This includes defining confidence tiers, validating interrupted tracts more carefully, and quantifying trade-offs between sensitivity and specificity across methods.

A standardized benchmarking framework would make downstream comparisons more robust and improve confidence when interpreting biological differences between taxa.

Extend statistical modeling of codon–length dynamics

The observed relationship between polyQ length and codon composition (%CAG vs %CAA) is one of the most promising findings of the project. Future analyses can expand this by testing additional robust models, evaluating non-linear behaviors across taxa, and incorporating covariates such as genome composition, proteome composition, or taxonomic grouping.

This would help clarify whether the different regression patterns observed across lineages reflect shared evolutionary mechanisms, lineage-specific constraints, or differences in repeat detection and sampling.

Final Remarks

This project began as an effort to study polyglutamine tracts across deuterostomes, but it ultimately became both an evolutionary analysis and a reusable computational framework for comparative repeat biology. By combining large-scale sequence screening, structured data storage, and cross-taxon statistical modeling, it revealed consistent links between polyQ length, codon composition, and lineage-specific patterns while also highlighting the importance of method choice in repeat detection (as we expected). More importantly, it established a foundation that can be extended to new repeat types, richer functional annotations, and broader phylogenetic questions. In that sense, the most valuable outcome is not only the results presented here, but the platform it creates for possible future discovery.
